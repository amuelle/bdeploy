The name of the process configuration, not to be confused with the name of the application launched by the process.
The start type of the process. This controls how a process can be started and whether it is considered vital for the instance.
  • INSTANCE: The process will be started/stopped when bulk controlling the instance, and is considered vital, i.e. the instance receives a warning marker if the process is not running.
  • MANUAL: The process will never be started automatically. A user needs to manually start the process.
  • MANUAL_CONFIRM: Same as MANUAL, except that the user will additionally be prompted to confirm starting the application.
Determines whether BDeploy tries to keep the process alive upon exit, restarting it after an increasing back-off period.
The amount of times BDeploy will try to restart a process which has Keep Alive enabled after the processes exited. The number of retries is reset once the process has been running for a certain amount of time.
The time that BDeploy will wait for the process to shut down once instructed to do so. If the process does not exit within this time, BDeploy will start to apply more force.
Determines whether the console in the UI should be attached remotely to the processes stdin stream. This allows to send commands to the process if the process supports this.
The main command executed by BDeploy when running the process.
