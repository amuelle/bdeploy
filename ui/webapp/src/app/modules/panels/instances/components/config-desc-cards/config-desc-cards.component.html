The name of the process configuration, not to be confused with the name of the application launched by the process.
The ID of the process, which is unique within the instance.
The main command executed by BDeploy when running the process.
The start type of the process. This controls how a process can be started and whether it is considered vital for the instance.
  • INSTANCE: The process will be started/stopped when bulk controlling the instance, and is considered vital, i.e. the instance receives a warning marker if the process is not running.
  • MANUAL: The process will never be started automatically. A user needs to manually start the process.
  • MANUAL_CONFIRM: Same as MANUAL, except that the user will additionally be prompted to confirm starting the application.
Determines whether BDeploy tries to keep the process alive upon exit, restarting it after an increasing back-off period.
The amount of times BDeploy will try to restart a process which has Keep Alive enabled after the processes exited. The number of retries is reset once the process has been running for a certain amount of time.
The time that BDeploy will wait for the process to shut down once instructed to do so. If the process does not exit within this time, BDeploy will start to apply more force.
Determines whether the console in the UI should be attached remotely to the processes stdin stream. This allows to send commands to the process if the process supports this.
Determines whether the application automatically starts upon system bootup.
The endpoints path. The final URI is constructed using by appending this path to the constructed base URI calculated from the rest of the endpoint configuration.
The port this endpoint is hosted on.
Whether to use HTTPS instead of HTTP.
Whether to trust all certificates.
If not trusting all certificates, gives an optional path on the server file system to a Java TrustStore holding required certificates to trust.
A passphrase for the configured Java TrustStore.
The authentication type to use when opening a connection to the endpoint.
If authentication is required, the user to send to the endpoint for authentication.
If authentication is required, the password to send to the endpoint for authentication.
The URL of the OpenID Connect token.
The ID of the OpenID Connect client.
The secret of the OpenID COnnect client.
The name of the instance.
A descriptive text for humans.
The intended purpose of the instance, hinting how critical changes to the instance can be.
The configured version of the product in use by this instance. The version dictates which applications are available, how to configure them, as well as the final binaries installed on the target system.
Whether all processes of start type INSTANCE will be started automatically when the BDeploy server starts.
Whether old versions of the instance will be uninstalled automatically. Versions older than the currently active version are eligible.
The internal ID of the configuration files. A change in this ID will show that any configuration file has changed.
The internal ID of the system used by this instance.
The version of the system in use. This changes when the system is modified.
